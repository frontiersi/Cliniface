Image Browser

Cliniface's database of images can be accesed from the File menu by selecting Image Browser or by pressing the TAB key. Upon startup, the existing image database is loaded which contains the paths to any 3DF files opened in Cliniface during the last session as well as any 3DF files discovered within directories added to the database. Upon first use (or if the database is empty at startup) the example images supplied with Cliniface will be added to the database and displayed. A thumbnail image of the subject from the selected row is shown in the bottom right of the dialog. Left double clicking a row in the table loads that image file or the selected file can be loaded by clicking the Load Selected Image button. Note that Cliniface allows a maximum of two images to be loaded at once.

Adding 3DF files to the database

Click Reset Database to clear the existing database of all image references. Note that this will only remove the file references from the database without deleting the files themselves. New 3DFs can be added to the database either by manually loading a 3DF file in the normal way from Cliniface's main menu, or by clicking the Add Directory button whereupon you will be asked to select a directory to recursively parse for 3DF files. Metadata from each discovered 3DF file is read and loaded into the database. Note that this may take a while if the selected directory contains a very large number of 3DF files, or if the directory is on non-local storage (e.g., on network attached storage).

Only 3DF files can be added to the database. 3D images in other file formats must first be saved to 3DF either by loading them individually in the normal way and then saving to 3DF format, or by using Cliniface's command line interface to convert the image to 3DF format.

Searching the image database

The bottom section of the dialog provides filters to restrict the displayed images to just those meeting the selected criteria. The Subject, Source, and Study filters accept plain text input as well as the wildcard characters _ or ? to represent a single unknown character, and the wildcard characters * or % to represent zero or more unknown characters. The Clear Filters button resets all filter fields to show all 3DF files in the database.

The information in each row shows from left to right the local path of the 3DF file, the date the image was taken, the subject identifier, the image source (if not blank), and the study identifier (if not blank) associated with the image. Every image is allocated a subject identifier if not provided starting with the string: _UNKNOWN_. Clicking the column headings in the table sorts the table contents by that field (subsequent clicks toggle the sort order).
